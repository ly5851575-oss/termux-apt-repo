# سياسة الأمان

هذا المشروع ينظم تثبيت أدوات دفاعية ولا يبدأ أي فحص تلقائي ضد أهداف خارجية.

## الإبلاغ

استخدم GitHub Security Advisories للإبلاغ الخاص، ولا تنشر بيانات حساسة داخل Issue عام.

## قواعد ثابتة

- لا تُرفع صور rootfs أو ملفات tar.xz أو ملفات إعداد خاصة أو تقارير حساسة.
- لا تُحذف أدوات موجودة عند إضافة أدوات جديدة؛ يُزال فقط الإدخال المكرر من قوائم الحزم.
- لا يوجد تكامل ذكاء اصطناعي ولا يتم إرسال بيانات إلى خدمات خارجية.
- أي فحص شبكي يجب أن يكون على هدف مصرح ومذكور في AUTHORIZED_TARGETS.txt.
- استخدم المصادر الرسمية والتحقق من TLS، ويفضل checksum أو signature عند توفرهما.
